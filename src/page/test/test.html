제 설명은 공식 문서를 요약해드리는 수준으로만 되어있습니다. 정확한 사용법을 익히기 위해서는 공식 문서를 반드시 읽어주세요.

이 프로젝트는 node.js에서 작성되었으므로 기존의 html과 다른 부분이 일부 있습니다. 각종 팁을 설명드리겠습니다.

  • URL을 등록할 수 있습니다. URL은 index.js 하단에서 확인할 수 있고, 어떤 html 파일을 띄울 지 결정할 수 있습니다.
  • 외부 파일 삽입 시, 파일 경로 설정이 온전하지 않습니다. src 폴더 안에서 돌아갈 수 있도록 하였으나, 기존의 경로 표현식과 다를 수 있습니다. 만약 외부 파일을 삽입하였는데 제대로 적용이 되지 않는 경우 크롬에서 F12를 눌러 경고문에서 경로가 어떻게 지정되어있는지 확인해보세요. 아무리 확인해봐도 잘 이해가 가지 않으시면 연락바랍니다.
  • HTML을 수정하고 저장하면 새로고침 시 새로운 코드가 반영되지만, node.js 부분은 수정하고 저장하면 즉시 반영되지 않습니다. 따라서 nodemon을 전역 설치하거나 서버를 껐다 켜주세요.
  • 본인이 구현한 프로그램을 앱에서 확인하고 싶으면 내부 ip를 통해 접속이 가능합니다.

부트스트랩은 클래스 기반의 CSS 라이브러리입니다. 부트스트랩을 사용하려는 웹 페이지에 CDN 서버로 부터 css파일 하나와 자바스크립트 하나를 삽입하면 됩니다. 파일 삽입 위치는 이 링크에 나온 순서와 정확히 일치해야 구동 시 문제가 없습니다. (잘 모르시겠으면 기존에 작성되어있는 코드를 그대로 복붙해서 메인을 지우면 됩니다.)

부트스트랩은 사전에 잘 정의되어있는 CSS 클래스의 집합이라고 이해하시면 편합니다. 아시다시피 CSS 클래스는 띄어쓰기를 기준으로 여러 개 부여할 수 있습니다.

Breakpoints

원문에서 확인하기

Breakpoints는 반응형에 관련된 기능입니다. 클래스 이름 뒤에 다음과 같은 접미어가 붙게되면 해당 범위 이상에서만 적용하겠다는 이야기 입니다.

Breakpoint Class infix Dimensions
X-Small None <576px
Small sm ≥576px
Medium md ≥768px
Large lg ≥992px
Extra large xl ≥1200px
Extra extra large xxl ≥1400px
예를 들어 클래스 이름이 hello-md라면 화면의 가로 길이가 768px 이상일 때 만 hello 기능을 넣겠다는 의미이고, hello-lg라면 화면의 가로 길이가 992px 이상일 때 만 hello 기능을 넣겠다는 의미입니다. 이는 모든 클래스에서 사용 가능하지 않고, 일부에서만 사용이 가능합니다. 앞으로의 설명이나 문서에서 위 접미어 sm, md, lg, xl, xxl을 만난다면 당황하지 않고 위 설명대로 해석하면 됩니다.

Grid

원문에서 확인하기

그리드는 부트스트랩에서 row와 col 클래스를 조합하여 가로 간격을 부여하는 기능입니다.

row는 영어 원문 뜻 처럼 가로 행의 역할을 하며, row 클래스는 행의 역할을 하기 위해 상위에 wrapper 형태로 들어가야 합니다. <div="row">...</div>의 형태로 들어가게 되며, 그 아래에는 col 클래스가 오게 됩니다.
<div="row">
    <div="col"></div>
    <div="col"></div>
    <div="col"></div>
</div>

col는 상위에 row가 있는 경우에 동작하게 된다는 점을 꼭 유의해야합니다. row 클래스 아래에 올 수 있는 col 클래스는 12등분이 가능합니다. 12등분은 col-숫자와 같이 표현합니다.
<div="row">
    <div="col-4"></div>
    <div="col-4"></div>
    <div="col-4"></div>
</div>
 <div="row">
    <div="col-3"></div>
    <div="col-3"></div>
    <div="col-3"></div>
    <div="col-3"></div>
</div>
 <div="row">
    <div="col-6"></div>
    <div="col-6"></div>
</div>
 눈치 채신분은 아시겠지만 col-숫자들의 숫자 합이 항상 12여야 합니다. 합이 12가 넘으면 다음 줄로 넘어가게 되며, 숫자가 없다면 남은 공간 만큼 간격이 균일하게 배정됩니다.

이 기능도 breakpoints 기능 사용이 가능합니다. 이 링크를 끝까지 꼭 정독해주시고 사용해주시면 됩니다.

Spacing

원문에서 확인하기

Spacing은 간격에 관한 기능입니다. 여러분이 CSS에서 학습했을 때 margin, padding을 아주 중요하게 배웠을텐데, 이를 아주 손쉽게 사용할 수 있도록 도와줍니다. margin의 약자는 m, padding의 약자는 p입니다. m이나 p 뒤에 어느 방향으로 줄 것인지를 적고, 접미사로 숫자를 1~5를 주면 됩니다.
방향은 s(start, 왼쪽), e(end, 오른쪽), t(top,위쪽), b(bottom,아래쪽), x(가로), y(세로), 아무것도 안쓰기(4방향) 입니다.
설명이 어려운 것 같으니 예시로 보여드리겠습니다.
(m 또는 p)(방향)-숫자
(예)
ms-2 : margin을 start (왼쪽)방향으로 2만큼 주겠다
px-3 : padding을 x(가로) 방향으로 3만큼 주겠다
m-5 : margin을 좌우상하 4 방향으로 5만큼 주겠다
py-1 : padding을 y(세로) 방향으로 1만큼 주겠다
mt-2 : margin을 t(상단) 방향으로 2만큼 주겠다
